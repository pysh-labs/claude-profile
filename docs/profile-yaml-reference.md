# profile.yaml reference

Complete reference for the `profile.yaml` schema consumed by `claude-profile new -f <file>`.

## Top-level keys

| Key | Required | Type | Description |
|---|---|---|---|
| `apiVersion` | yes | string | Must equal `claude-profile.io/v1`. |
| `kind` | yes | string | Must equal `Profile`. |
| `metadata` | yes | object | Profile identity. |
| `statusline` | yes | object | Visual identifier in the rendered statusline. |
| `marketplaces` | no | map | Plugin marketplaces to register. |
| `plugins` | no | list | Plugins to install (`name@marketplace`). |
| `mcp_servers` | no | map | MCP servers written to `.mcp.json`. |
| `settings_overrides` | no | map | Deep-merged into `settings.json` after our defaults. |
| `claude_md` | no | string | Inline content for `CLAUDE.md`. |

## `metadata`

```yaml
metadata:
  name: personal           # required
  description: ...         # optional, free text
```

## `statusline`

```yaml
statusline:
  label: personal          # required, short prefix shown in the statusline
  color: green             # required, one of: red, green, yellow, blue, magenta, cyan, white
```

## `marketplaces`

Translated into `extraKnownMarketplaces` in `settings.json`.

```yaml
marketplaces:
  claude-plugins-official:
    type: github           # required, currently only "github"
    repo: anthropics/claude-plugins-official     # required
```

## `plugins`

```yaml
plugins:
  - atlassian@claude-plugins-official
  - superpowers@claude-plugins-official
```

Format: `<plugin-name>@<marketplace-name>`. The marketplace must be declared under `marketplaces`. Each plugin is installed by invoking `claude plugin install <plugin>` in a subprocess with `CLAUDE_CONFIG_DIR=<target>` set. Installs are sequential; failures of one plugin do not abort the others.

## `mcp_servers`

```yaml
mcp_servers:
  atlassian:
    command: docker        # required
    args: [run, -i, --rm, mcp/atlassian]
    env:
      USER: ${USER_NAME}   # ${VAR} interpolated at init-time
```

The `${VAR}` syntax is expanded from the host environment when `init` runs. A missing variable causes `init` to fail before any files are written.

## `settings_overrides`

Deep-merged on top of the keys generated by `claude-profile`. Merge semantics:

- Objects merge recursively
- Arrays and scalars **replace** (no concat, no union)
- On conflict, the value from `settings_overrides` wins

```yaml
settings_overrides:
  skipDangerousModePermissionPrompt: true
  permissions:
    allow: ["Bash(az *:*)"]
```

## `claude_md`

Plain string, written verbatim to `CLAUDE.md`. v0.1 supports inline content only; file includes and overlay rendering are deferred to v0.2.

```yaml
claude_md: |
  Branch prefix: feature/, bugfix/, hotfix/.
  Always include ticket number.
```
